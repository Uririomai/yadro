Я бы организовал тестирование по принципу пирамиды тестов с 3 основными уровнями:
1. Юнит тесты:
    - компоненты
    - стейт 
    - различные функции
2. Интеграционные тесты:
    - взаимодействие компонентов
    - Работа со стейтом
    - комплексные сценарии
3. Е2Е тесты:
    - полные сценарии
    - работа с api

Для тестирования использовал бы Jest + React Testing Library + redux-mock-store для тестов с Redux

Для отчетности:
- Jest HTML Reporter для юнит-тестов
- Cypress Dashboard для E2E тестов
- Codecov для отслеживания покрытия
